# Paper Text Blocks

This folder stores reusable text blocks that can later be adapted into the paper, appendix, or repository documentation.

These files are not final paper sections. They are intermediate writing artifacts used to preserve methodological decisions, experiment summaries, and interpretation notes.

## Current Files

| File | Purpose |
|---|---|
| `toy_pipeline_validation.md` | Summarizes the toy example used to validate the normalization and evaluation pipeline. |

## Usage

Text blocks in this folder may be reused in:

- methodology section;
- experimental protocol section;
- appendix;
- artifact documentation;
- supervisor/advisor reports.

## Reproducibility Note

When a text block summarizes results, it should mention the corresponding scripts and output folders used to generate those results.

| File | Purpose |
|---|---|
| `methodology_benchmark_framework.md` | Describes the full benchmark methodology, including ground truth design, prompt conditions, normalization, evaluation, distances, cost tracking, and aggregation. |

| `paper_positioning.md` | Defines the paper positioning, contribution, research questions, and venue framing. |

| `expected_analysis_plan.md` | Defines how aggregate benchmark results should be analyzed and interpreted. |
